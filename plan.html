Important Notices

Before We Start

Critical items that affect setup and deployment.

100% Spark Free Plan (No Credit Card Required)

By storing bill photos as compressed Base64 strings in Firestore and using pre-set vector male/female avatars, we avoid using Firebase Storage entirely. This keeps the app completely free and runs on the Spark tier.

Vercel Hobby Plan — Non-Commercial Only

The free Vercel plan is for personal projects. If this becomes commercial, you'll need Vercel Pro ($20/mo). For a personal tracking app between two people, Hobby is perfect.

Real-Time Multi-Device Sync

Using Firestore's onSnapshot listeners, changes made on Num's phone will instantly appear on Kaew's device (and vice versa). No manual refresh needed.

Security

Security Architecture

Every layer hardened — from cookies to Firestore rules.

Authentication

  • Firebase Anonymous Auth per device
  • PIN hashed with bcryptjs (salt rounds: 10)
  • Max 5 PIN attempts, then 30s lockout
  • Auto-logout after 7 days of inactivity

Session Cookies

  • httpOnly — can't be read by JavaScript
  • secure — HTTPS only in production
  • sameSite: lax — CSRF protection
  • Server-side verification on every request

Firestore Rules

  • Authenticated users only (no public reads)
  • Write validation: required fields check
  • User can only modify own device data
  • Server-side createdAt timestamps only

Image Handling

  • Client-side canvas compression to < 80KB
  • Encoded as Base64 JPEG strings
  • Stored directly in Firestore document
  • Validated on client side: type and size

Data Privacy

  • No personal data sent to 3rd parties
  • PIN never stored in plain text
  • Firebase environment keys in .env.local
  • Admin SDK on server only (never in client bundle)

Input Validation

  • All amounts validated (positive numbers only)
  • XSS prevention: sanitized text inputs
  • Decrease can't exceed current balance
  • Server-side validation in API routes

App Screens

Every page with routing, features, and UX details.

Login Page

/login

Profile selection showing Num & Kaew (and any future profiles) as avatar cards. Tap a profile to reveal an animated PIN keypad. Handles wrong PIN with shake animation and lockout after 5 attempts. Cookie set on success for 7-day persistence. "Create New Profile" option at bottom.

Profile Cards PIN Keypad Cookie Session Lockout Protection Create Profile

Main Dashboard

/ (home)

Big animated circle at center showing total balance in ฿. Ring color changes: green when people owe you, red when you owe. SVG stroke animation with counting number effect. Below: two large tactile buttons — "Add" and "Paid Back". Recent activity feed (last 5 entries) with real-time updates via Firestore onSnapshot.

Balance Circle (SVG) Real-time Sync Quick Add Quick Paid Back Activity Feed Counter Animation

Profile Page

/profile

Shows who is currently logged in (Num or Kaew). Large avatar card with stats (total lent, total borrowed, net balance). Switch between profiles. Edit: change name, select avatar (Male / Female outline icon), pick accent color. All profiles listed with switch option.

Profile Card Avatar Selector (M/F) Color Picker Switch User Personal Stats

Transaction History

/transactions

Full audit log — shows who did what, how much, and when. Every add, edit, and payback is logged with the profile name of who made the change. Filter by person, action type, or date range. Color-coded: green for paybacks, red for new borrows. Expandable cards show edit history and bill photo thumbnails.

Audit Trail Who Edited Filters Color Coded Bill Thumbnails Date Range

Add / Edit / Decrease

/records/new · /records/[id]/edit · /records/[id]/decrease

Add: Person name, amount (฿), type (borrow/lend), note. Optional bill photo (compressed to Base64). Edit: Modify description or amount — logged in audit trail. Decrease (Paid Back): Enter repayment amount with real-time balance preview. Can't exceed current balance. Animated progress bar shows remaining debt.

Project Setup & Design System Phase 1

Next.js 15 init with TypeScript. Firebase project setup. globals.css with all design tokens. PWA manifest. Install dependencies.

~30 min

Auth & Security Phase 2

Firebase Anonymous Auth. Profile creation. PIN keypad with bcrypt hashing. Cookie sessions (httpOnly, secure, sameSite). Middleware route protection. Lockout logic. Firestore security rules.

~50 min

Main Dashboard Phase 3

Large SVG balance circle with stroke animation. Counter animation for ฿ amounts. Quick action buttons. Recent activity feed with real-time Firestore listeners.

~45 min

CRUD: Add / Edit / Paid Back Phase 4

MoneyForm component (shared). Add new record with validation. Edit with audit trail. Paid Back flow with balance preview. Server-side validation in API routes.

~50 min

Transaction History Phase 5

Chronological audit log. Filter by person/type/date. Expandable cards with edit history. Color-coded entries. Real-time updates.

~35 min

Profile & Avatar Phase 6

Profile management page. Vector avatar selector (Male/Female icons) and accent color picker. Profile switching. Stats overview.

~25 min

Bill Photo Handling Phase 7

Camera capture or file picker. Client-side canvas compression down to < 80KB. Convert to Base64 string and save directly in Firestore. Thumbnail preview in history.

~30 min

Navigation, Toast & Polish Phase 8

Bottom navigation with animated FAB. Header component. Toast notification system. Loading skeletons. Page transitions. Responsive testing.

~35 min

PWA, Service Worker & Deploy Phase 9

Service worker caching (static assets, offline fallback). PWA install prompt. Deploy to Vercel. Final QA on mobile + desktop.

~25 min
